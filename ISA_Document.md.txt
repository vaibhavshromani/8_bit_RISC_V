# Control Signal Truth Table

One row per instruction, exactly as the project guide recommends drawing
before writing the control unit code.

| Opcode | Instr | reg_write | mem_write | mem_read | alu_src | mem_to_reg | alu_op |
|--------|-------|-----------|-----------|----------|---------|------------|--------|
| 000    | ADD   | 1         | 0         | 0        | 0       | 0          | 00     |
| 001    | SUB   | 1         | 0         | 0        | 0       | 0          | 01     |
| 010    | AND   | 1         | 0         | 0        | 0       | 0          | 10     |
| 011    | OR    | 1         | 0         | 0        | 0       | 0          | 11     |
| 100    | LOAD  | 1         | 0         | 1        | 1       | 1          | 00     |
| 101    | STORE | 0         | 1        | 0        | 1       | 0          | 00     |

## Signal meanings

- **reg_write** → 1 if the instruction writes a result back to the register file  
- **mem_write** → 1 for STORE (writes ALU-computed address/data into data memory)  
- **mem_read** → 1 for LOAD (data memory is combinationally readable in this design)  
- **alu_src** →  
  - `0` = ALU operand B comes from register file (R-type)  
  - `1` = ALU operand B comes from immediate (LOAD/STORE address calculation)  
- **mem_to_reg** →  
  - `0` = write-back from ALU result  
  - `1` = write-back from memory (LOAD only)  
- **alu_op** → selects ALU operation:  
  - `00` = ADD  
  - `01` = SUB  
  - `10` = AND  
  - `11` = OR  
  - LOAD/STORE always use ADD (base + immediate)
